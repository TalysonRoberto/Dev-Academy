## Introdução

Tags semânticas descrevem o **significado** do conteúdo, não apenas a aparência. Melhoram acessibilidade e SEO.

## Conceito Explicado

```html
<header>   <!-- Cabeçalho da página ou seção -->
<nav>      <!-- Menu de navegação -->
<main>     <!-- Conteúdo principal (único por página) -->
<section>  <!-- Seção temática -->
<article>  <!-- Conteúdo independente -->
<aside>    <!-- Conteúdo lateral -->
<footer>   <!-- Rodapé da página ou seção -->
```

## Exemplo Visual

**Com `<div>` (sem semântica):**
```html
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">...</div>
<div class="footer">...</div>
```

**Com tags semânticas:**
```html
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>
```

O navegador e leitores de tela entendem a estrutura com tags semânticas.

## Boas Práticas

- ✅ Use `<main>` apenas uma vez por página
- ✅ Use `<nav>` para menus de navegação
- ✅ Use `<article>` para conteúdo independente (posts, cards)
- ❌ Não use `<div>` quando existe tag semântica
- ❌ Não use `<section>` sem título dentro

## Atividade Final

Refatore a página fornecida:
1. Substitua `<div class="header">` por `<header>`
2. Substitua `<div class="content">` por `<main>`
3. Substitua `<div class="footer">` por `<footer>`
4. Adicione `<section>` dentro do main se apropriado
